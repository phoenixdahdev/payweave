## Initialize â€” the provider is narrowed at compile time

```ts
import { createPaystack, createFlutterwave } from "payweave"

const paystack = createPaystack({ secretKey: process.env.PAYSTACK_SECRET_KEY! })
const flutterwave = createFlutterwave({ secretKey: process.env.FLW_SECRET_KEY! }) // v3 by default

paystack.provider // "paystack"
paystack.environment // "test" | "live" â€” inferred from the key prefix
// paystack.flutterwave  ->  compile-time error: property does not exist
```

Test vs live is a **config change, not a code change** â€” it is inferred from the key prefix:

| Provider | Test prefix | Live prefix |
| --- | --- | --- |
| Paystack | `sk_test_` | `sk_live_` |
| Flutterwave v3 | `FLWSECK_TEST-` | `FLWSECK-` |

A malformed or unrecognized key throws `PayweaveConfigError` at construction, naming the expected prefixes â€” a request is never sent with a key that can't be classified.

</Step>

## Your first checkout (unified layer)

The unified layer always takes **integer minor units**; the adapter converts to each provider's native units for you.

```ts
const sdk = createPaystack({ secretKey: process.env.PAYSTACK_SECRET_KEY! })

const checkout = await sdk.unified.checkout.create({
  amount: { value: 500_000, currency: "NGN" }, // â‚¦5,000 in kobo
  customer: { email: "ada@example.com" },
  reference: "order_8123", // â†’ Paystack `reference` / Flutterwave `tx_ref`
  redirectUrl: "https://app.example.com/pay/callback",
})

// â†’ { checkoutUrl, reference, providerRef, raw }
console.log(checkout.checkoutUrl)
```

Redirect your customer to `checkoutUrl`. After they return, verify before granting value:

```ts
const result = await sdk.unified.verify({ reference: "order_8123" })
// â†’ { status: "success" | "failed" | "pending" | "abandoned" | "reversed",
//     amount: { value, currency }, customer, paidAt, channel, raw }

if (result.status === "success") {
  // fulfil the order
}
```

Swap `createPaystack` for `createFlutterwave` and the exact same unified code keeps working.
