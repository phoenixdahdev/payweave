`payweave init` — interactive setup wizard (PW-1005, docs/v1/cli.md §1).

- `npx payweave init` asks which provider(s) you're using (Stripe, Paystack, Flutterwave — multiselect, ≥1 required) and which database (Prisma, Drizzle, Postgres, MySQL, SQLite/libSQL, MongoDB, or none for a payments-only project), detects your framework from `package.json` deps (+ a `next.config.*` filesystem-marker fallback for Next.js), and scaffolds: a `payweave.ts` config (named `payweave` export, matching PW-1002's discovery contract) wired to a database + `products` only when a database is chosen (an unconditional `products` without a `database` is a `PayweaveConfigError` at construction, unified-config.md §2 rule 5); an example `products.ts` (`free`/`pro` plans); a framework-specific webhook route (Next.js App Router, Express, Fastify, or a plain `node:http` fallback) calling `payweave.webhooks.constructEvent` on the untouched raw body; `.env.example` with env var *names* only, never values; a Prisma schema fragment or a Drizzle guidance file (re-exporting the real, already-shipped `payweave/db/drizzle` `pgSchema`/`mysqlSchema`/`sqliteSchema`) for those two database choices; and an optional `lib/payweave-client.ts` frontend `fetch` helper.
- Every interactive decision goes through an injectable `InitPrompts` seam (mirrors `push.ts`'s `confirm` seam) — the real CLI uses `@clack/prompts` (now bundled into `dist/cli` via `tsup.cli.config.ts`'s `noExternal`, alongside its small transitive chain: `@clack/core`, `sisteransi`, `fast-wrap-ansi`, `fast-string-width`, `fast-string-truncated-width` — all pure JS/ESM, `scripts/check-cli-deps.mjs` verified green); tests drive the wizard non-interactively by injecting their own `InitPrompts`.
- Overwrite semantics: per-file confirmation prompt before clobbering an existing file; `--force`/`-f` skips every prompt and always overwrites. Declining (or running non-interactively with no injected prompt seam, where prompting isn't possible) leaves that file untouched and makes the overall command exit non-zero with a clear summary naming the conflicting file(s) and pointing at `--force` — files with no conflict still get written.
- `postgres`/`mysql`/`mongodb`/`prisma` database choices wire the exact import + factory-call shape `database.md` §1 documents even though those adapters are still placeholder stubs pending PW-704/705/709/707 — the generated code is correct ahead of the adapters landing.
