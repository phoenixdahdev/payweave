Provider adapter contract v2 (PW-608, providers.md §4): `ProviderAdapter` gains `readonly configKey: string` and `configSchema: ZodType` — the extension point that lets a third-party adapter register onto `createPayweave`'s root config by config key alone, with zero edits to `core/provider.ts` or `core/config.ts` (proved by a toy `"acme"` adapter in `test/core/adapter-contract.test-d.ts`). `webhooks` gains `signatureHeader: string` (the multi-provider dispatcher's detection key, unified-config.md §5), and the contract gains an optional, unimplemented `billing` slot (typed placeholder for PW-803/804). `defineProvider`'s runtime validation now asserts all three new fields. All existing v1 `ProviderAdapter` fields are unchanged.

All three providers now export a v2 adapter descriptor alongside their existing `*Client` classes: `paystackAdapter` (`src/paystack/client.ts`), `flutterwaveAdapter` / `createFlutterwaveAdapter(version)` (`src/flutterwave/client.ts` — one definition parameterized by `"v3" | "v4"`, per AGENTS.md §11's version isolation), and `stripeAdapter` (`src/stripe/client.ts`, reusing PW-601's `stripeHttpOptions` so its transport is byte-identical to the real wiring). These are additive metadata alongside `createPayweave`'s existing direct construction path in `src/index.ts`, which is unchanged — every provider's Surface A / unified / webhook behavior is identical to before this change.

Webhook dispatch (`src/webhooks/index.ts`'s `constructEvent`) now normalizes Stripe events through the same `unified/mappings.ts` tables (`STRIPE_EVENT_MAP` / `STRIPE_EVENT_STATUS_SPLIT_MAP`, landed by PW-607) every other provider already used, instead of the previous hardcoded `unifiedType: "unknown"`. A mapped Stripe event (e.g. `payment_intent.succeeded`) now normalizes to its real unified type (`payment.succeeded`); an unmapped event still falls through to `"unknown"`, exactly like Paystack/Flutterwave. `unified/mappings.ts` itself is untouched — this change only wires an existing table into a call site that was bypassing it.
