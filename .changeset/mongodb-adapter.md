Land the MongoDB `DatabaseAdapter` (database.md §4, PW-709) at `payweave/db/mongodb` — first-party v1, not a bolted-on community adapter. `mongodbAdapter({ url, dbName } | { client, dbName })` accepts an eagerly-validated `mongodb://`/`mongodb+srv://` connection string (driver imported and connected lazily) or an already-constructed `MongoClient` you own. Documents are the `src/db/schema.ts` row shapes verbatim, with `id` stored as `_id` (`pw_webhook_events.dedupeKey` is `_id` directly, its own documented natural-key exception). `balances.consume` and `webhookEvents.claim` are each exactly ONE `findOneAndUpdate` with `upsert: true` and an aggregation-PIPELINE update — the lazy period reset, conditional decrement, and once-only claim/steal logic all run atomically server-side in a single round trip, on any topology; the period math matches `src/products/period.ts`'s `currentPeriod()` via `$dateAdd`/`$dateToParts`, anchor-relative, never iterating on previously clamped outputs. `migrations.status()`/`apply()` do NOT use the SQL migrations engine (mongo has none) — they idempotently ensure every `pw_*` collection's indexes, including the active-subscription rule as a unique index with `partialFilterExpression`. `transaction()` attempts a real multi-document transaction and falls back to the documented non-atomic path on standalone deployments (letting MongoDB's own error be the authority, not a topology probe) — safe because the two hot paths with a hard atomicity requirement are already atomic per document regardless of topology. `mongodb` is an optional peerDependency (+ devDependency) — `payweave`'s runtime `dependencies` stay zod-only, and the root package never imports the driver. `payweave/db/mongodb` no longer throws the PW-505 placeholder error. Conformance registers in BOTH standalone and single-node replica-set modes, gated behind `PW_DB_CONFORMANCE_DOCKER` for PW-710's CI matrix (no Docker in this environment). `src/core/redact.ts` now also scrubs credential-bearing `mongodb://`/`mongodb+srv://` connection strings.
