Stripe Surface A adds its money-return and endpoint-management modules (PW-605): `payweave.stripe.refunds` (create, retrieve, update, cancel, list/iterate) and `payweave.stripe.webhookEndpoints` (create, retrieve, update, delete, list/iterate). Refund creation targets a `charge` or `payment_intent` (or `origin: "customer_balance"`) — enforced before any network call — and accepts an `idempotencyKey` option wired to Stripe's `Idempotency-Key` header (which also makes the money-moving POST retry-eligible; bare POSTs are never auto-retried). `refunds.cancel` is only valid for `requires_action` refunds, per the Stripe docs. `webhookEndpoints.create` is the ONLY response carrying the endpoint's `whsec_*` signing secret ("Only returned at creation") — the response schema names `secret` so callers can capture it once, and core `redact` masks it from all logs. Requests are validated with Zod before the form encoder runs and go to the wire as `application/x-www-form-urlencoded` bracket notation (`enabled_events[0]=...`); responses are loose bare resources (drift is logged, never thrown). Lists paginate with `starting_after`/`has_more` cursor iterators.
